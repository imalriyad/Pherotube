Discuss the scope of var, let, and const

var

In Javascript we can declare a variable in three way var let const

Those variable are declared with var are hoisted to the top of their containing function or global scope, which means we can use them before they are declared, but they will have an initial value of undefined.

Var is redeclareable

let

Variables declared with let have block scope, which means they are only accessible within the block

We cannot redeclare a variable using let in the same block scope.

const

Variables declared with const have block scope, which means they are only accessible within the block

We cannot redeclare a variable using const in the same block scope.

Tell us the use cases of null and undefined

null:

Null means an empty value or a blank value.

We can assign null to a variable or property to indicate that it intentionally has no value

Example: let x = null

undefined:

Undefined means the variable has been declared, but its value has not been assigned.

When we declare a variable but don't assign a value to it, it is automatically initialized to undefined

Example: let x;

What do you mean by REST API?

REST stands for Representational State Transfer

API stands for Application programming interface

What are the HTTP methods supported by REST?

1. GET : A method of retrieving data from the server.
2. POST: Data is sent to the server in order to create a new resource.
3. PUT: Updates an existing resource or creates a new one if it does not exist.
4. DELETE: This request is used to remove a resource from the server.
